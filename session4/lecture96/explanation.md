# 백준 16637 — 괄호 추가하기 (재귀 + 누적합 풀이 해설)

## 1. 문제의 핵심 조건

- 정수는 0~9, 연산자는 `+`, `-`, `*`
- 연산자 우선순위는 **모두 동일**, 좌→우 순차 계산
- 괄호 안에는 연산자가 **정확히 1개**
- 괄호는 **중첩 불가**, 서로 **겹칠 수도 없음**

이 조건들이 합쳐지면 결과적으로 모든 괄호는 `(정수 연산자 정수)` 형태로, 서로 disjoint한 위치에만 들어갈 수 있다.

## 2. 알고리즘 아이디어

좌→우 순차 계산 + 괄호 비중첩 두 조건 덕분에, 각 연산자 위치에서 결정해야 할 것은 **"오른쪽 피연산자를 단일 숫자로 쓸까, 아니면 다음 두 숫자를 괄호로 묶어서 한 덩어리로 쓸까"** 두 가지뿐이다.

이전 계산 결과를 `_num` 하나에 누적해서 들고 다니면 그 뒤로 어떤 분기를 타든 정확히 이어 계산된다. 이게 "누적합" 패턴이다.

## 3. 변수의 의미 ([beackjoon_16637.cpp:19](beackjoon_16637.cpp#L19))

```cpp
void go(int here, int _num)
```

- `here`: `num[0]..num[here]`까지 누적 계산을 끝낸 상태. 이제 처리할 연산자는 `oper_str[here]`.
- `_num`: `num[0]..num[here]`까지의 누적 결과값.

여기서 `oper_str[k]`는 `num[k]`와 `num[k+1]` 사이의 연산자이다.

종료 조건: `here == num.size() - 1` (마지막 숫자까지 누적 완료)

## 4. 두 가지 분기

**Case 1 — 괄호 없음 (한 칸 전진)** ([beackjoon_16637.cpp:24](beackjoon_16637.cpp#L24))

```
새_num = _num [oper_str[here]] num[here+1]
go(here+1, 새_num)
```

**Case 2 — 다음 두 숫자를 괄호로 묶음 (두 칸 점프)** ([beackjoon_16637.cpp:26-29](beackjoon_16637.cpp#L26-L29))

```
temp   = num[here+1] [oper_str[here+1]] num[here+2]
새_num = _num [oper_str[here]] temp
go(here+2, 새_num)
```

## 5. 예제 1: `8*3+5` (정답 64)

### 준비

```
index:   0   1   2
num:     8   3   5
            ↑   ↑
oper:      [0] [1]
            *   +
```

`go(0, 8)`로 시작. 종료 조건은 `here == 2`.

### 재귀 트리

```
                go(0, _num=8)
               /              \
          Case 1                Case 2
       (8*3 진행)            ((3+5)로 묶음)
       한 칸 전진             두 칸 점프
          ↓                       ↓
      go(1, 24)               go(2, 64)
      "8*3" 누적              "8*(3+5)" 누적
          ↓                       ↓
       Case 1                  종료
       (24+5)                  ret=64 ✓
          ↓
      go(2, 29)
       ret=29
```

### 각 leaf가 만드는 식

| 경로 | 식 | 결과 |
|------|-----|------|
| Case1 → Case1 | `8*3+5` | 29 |
| Case2 | `8*(3+5)` | **64** ✓ |

`max = 64`

### 정답 경로 단계별 추적

**Step 1**: `go(0, _num=8)`
- `oper_str[0]='*'` 처리 차례
- **Case 2 선택**: 다음 두 숫자를 괄호로 묶기
  - `temp = num[1] + num[2] = 3+5 = 8`
  - `새_num = 8 * 8 = 64`
  - 호출: `go(2, 64)` ← here 2칸 점프

**Step 2**: `go(2, 64)`
- `here == num.size()-1 == 2` → 종료
- `ret = max(ret, 64)` ✓

## 6. 예제 2: `8*3+5+2` (정답 66)

### 준비

```
index:   0   1   2   3
num:     8   3   5   2
            ↑   ↑   ↑
oper:      [0] [1] [2]
            *   +   +
```

`go(0, 8)`로 시작. 종료 조건은 `here == 3`.

### 재귀 트리

```
                         go(0, _num=8)
                        /              \
                  Case 1                Case 2
              (8*3 진행)            ((3+5)로 묶음)
              한 칸 전진             두 칸 점프
                ↓                       ↓
            go(1, 24)               go(2, 64)
            "8*3" 누적              "8*(3+5)" 누적
            /        \                 ↓
       Case 1       Case 2          Case 1
      (24+5)      ((5+2)묶음)       (64+2)
        ↓             ↓                ↓
   go(2, 29)     go(3, 31)         go(3, 66)
        ↓         ret=31 ✓          ret=66 ✓
     Case 1
      (29+2)
        ↓
   go(3, 31)
    ret=31 ✓
```

### 각 leaf가 만드는 식

| 경로 | 식 | 결과 |
|------|-----|------|
| Case1 → Case1 → Case1 | `8*3+5+2` | 31 |
| Case1 → Case2 | `8*3+(5+2)` | 31 |
| Case2 → Case1 | `8*(3+5)+2` | **66** ✓ |

`max = 66`

### 정답 경로 단계별 추적

**Step 1**: `go(0, _num=8)`
- `oper_str[0]='*'` 처리 차례
- **Case 2 선택**: 다음 두 숫자를 괄호로 묶기
  - `temp = num[1] + num[2] = 3+5 = 8`
  - `새_num = 8 * 8 = 64`
  - 호출: `go(2, 64)` ← here 2칸 점프

**Step 2**: `go(2, 64)`
- 의미: 식 `8*(3+5)`까지 계산해서 64를 들고 있음
- `oper_str[2]='+'` 처리 차례
- **Case 1 선택**: 그냥 다음 숫자 하나
  - `새_num = 64 + 2 = 66`
  - 호출: `go(3, 66)`

**Step 3**: `go(3, 66)`
- `here == num.size()-1 == 3` → 종료
- `ret = max(ret, 66)` ✓

---

## 7. 핵심 통찰

- **Case 1 = 한 칸 전진** → 다음 숫자 하나만 먹기
- **Case 2 = 두 칸 점프** → 다음 두 숫자를 괄호로 묶어서 한 덩어리로 먹기

각 연산자 위치에서 이 이진 선택만 하면 되므로, 가능한 모든 비중첩 괄호 조합이 자연스럽게 탐색된다.

`_num`이 좌측 누적값을 그대로 들고 가는 게 가능한 이유는 **괄호 밖이 좌→우 순차 계산**이기 때문이다. 만약 우선순위가 다르거나 괄호 중첩이 가능했다면 이 누적이 깨져서 더 복잡한 DP가 필요했을 것이다.

### 분기 개수의 의미

길이 N인 수식의 숫자 개수를 k = (N+1)/2 라 하면, 연산자 개수는 k-1. 각 연산자 자리에서 두 분기 중 하나를 선택하므로 단순 상한은 `2^(k-1)`이지만, Case 2가 두 칸 점프하면서 다음 연산자 자리를 건너뛰므로 실제로는 피보나치 수에 가까운 분기 수로 감소한다. N ≤ 19에서 충분히 빠르게 동작한다.
