# 인접행렬 vs. 인접리스트

### 공간 복잡도

V = Vertax(정점) 개수
E = Edge(간선) 개수

- 인접행렬 : O(V^2)
- 인접리스트 : O(V + E) : 연결된 간선의 정점만 포함되어야 하기 때문!

### 시간복잡도

#### 간선 한개 찾기

- 인접행렬 : O(1)
- 인접리스트 : O(V)

#### 모든 간선 찾기

- 인접행렬 : O(V^2)
- 인접리스트 : O(V + E)

## 언제? 무엇을 쓰나?

- 그래프의 연결이 띄엄띄엄할때는 -> 인접리스트 (메모리 효율화)
- 그래프의 연결이 조밀할때는 -> 인접행렬

- 보통은 인접리스트를 써라
  - 연결이 띄엄띄엄 한 문제들이 많다
  - 배열은 크기가 너무 크면, 한계가 있다.
- 문제 풀때, 인접행렬이 주어지면 그냥 인접행렬로 푼다
