# 강의 해설을 보지 않고 생각해보기

일단 유형이 뭔지는 아니가, 투포인터 개념만 가지고 생각을 해본다!

투포인터의 개념이 두개의 포인터를 가지고 목표지점을 찾는다.

- 정렬을 시킨다.
- 시작점과 끝점에 각각 포인터를 한개씩 지정한다.
- 우선 두 수를 더해보고, X와 값이 같으면 결과카운팅을 한개씩 올린다. 그리고 두 포인터를 각각 한칸씩 옮긴다.
- 두 수를 더해보고 X와 값이 같지 않다면, 시작점 포인터를 옮기고 두수를 더해본다.
- 이 작업을 포인터가 서로 교차되거나 겹칠때까지 계속 한다.

# 강의 해설

- 작은수 부터 정렬을 시킨다. (오름차순으로)
- 양 끝 쪽에 포인터를 각각 둔다
- 두 수를 더했을 때, X 보다 크거나 같으면, 오른쪽 포인터를 왼쪽으로 (더 작은수가 되도록)
- 두 수를 더했을 때, X 보다 작으면 왼쪽 포인터를 오른족으로 (더 큰수가 되도록)

# 이 알고리즘의 시간복잡도

O(N \* logN)
