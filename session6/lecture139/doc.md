# 그리디의 조건

그리디 알고리즘으로 문제를 풀기전에 다음과 같은 조건이 충족이 되어야한다
2가지가 충족되어야 한다

1. 최적 부분 구조
   문제를 작은 문제로 나눌 수 있을 때!

2. 탐욕 선택 속성
   각 단계에서 최적의 선택이 전체 문제에 대해서도 최적해를 찾는데 도움을 잔다.

그리디가 막힐때 대표 예시
12100원 을 최소의 갯수로 만들어라

10000 5개
5000 5개
1000 5개
100 5개

-> 명제 : 가장 큰! 지폐부터 쓴다.
-> 10000 1개 + 1000 2개 + 100 1개

그런데 이 문제를 만약 아래와 같이 바꾼다면

8000 5개
5000 5개
1000 5개
100 5개

-> 기존 명제 : 가장 큰! 지폐부터 쓴다.
-> 8000 1개 + 1000 4개 + 100 1개
-> 하지만 최적해는 5000 2개 + 1000 2개 + 100 1개

이때는 기존의 명제 `가장 큰! 지폐부터 쓴다.` 는 틀린 명제가 된다.


명제 -> 증명

- 직접 증명
- 간접 증명
  - 귀류법
    - 명제 -> 거짓 이라 가정하고
    - 거짓이 틀렸다라는 걸 증명하는 방법

현실적인 방법

- 어떤 명제를 생각한 다음
- 최적해는 무엇일까 생각하고 코드로 구현
- 해당 명제가 틀리면, 빠르게 명제를 변경해서 풀어가야한다.

무식하게 풀다가
-> DP 로 풀어보고
-> 그리디로 푼다

- 우선순위 Q를 쓰면서
