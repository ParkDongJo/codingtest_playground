# 강의 해설 보기 전

일단 다 잘라내고, 그리디로 푼다는 가정하에!

- 입력 받은 시간대에서 가장 처음에 시작하는 시간대와, 가장 마지막에 도착하는 시간대를 구한다.
- 가장 끝시간 - 가장 앞선 시간 을 구한 범위에서 가장 적은 시간을 쓰는 사람의 스케줄부터 범위에서 빼간다

# 강의 해설

-> 우선! 무식하게 푼다!!!

무식하게 풀때, 문제의 조건을 봤을 때~ 성능상 통과하지 못할거 같다는 판단이 든다면!

-> DP 로 푼다!!
그리디로 푸는 문제 대부분은 DP로도 풀수 있는 경우가 많다.
하지만,
메모이제이션 기법을 함께 사용하는데, 메모이제이션 해야하는 경우의 수가 1000만 이상 되거나 할때!! 그때는 그리디로 넘어가야한다.

-> DP가 안되면 그리로 푼다!!
그리디를 푸는 방법은 명제를 몇가지 세운다! 그리고 그 명제의 반례가 없는지 가볍게 체크한다. 반례가 생긴다면 빠르게 다음으로 넘어간다.

들어오는 벡터를 정렬

- start 기준으로 오름차순 해보기
  - 반례가 있었음
  - 대부분 반례는 사례를 통해 확인할수 있겠지만, 그렇지 않다면 본인이.. 직접 찾아야 하는거 같음
- end 기준으로 오름차순 해보기
- 구간의 길이를 기준으로 오름차순 해보기
