# 동적계획법 DP

문제를 해결하기 위해 문제를 작은 하위 문제로 나누고, 그 결과를 저장(메모라이제이션)하여 풀어가는
-> 덕분에 하위문제를 반복해서 풀지 않는다.

## DP 의 조건

1. 참조 투명성을 가져야한다.

- 외적 요소(변수)에 의존하게 되면 안다.

2. 최적 부분 구조

- 작은 하위 문제들을 이용해 문제를 해결할 수 있는 경우
- 좋은 예시 ) 피보나치 수열

3. 겹치는 부분 문제

- 동일한 하위 문제가 여러번 반복해서 등장하는 경우

4. 방향성이 있고 사이클이 없는 그래프 구조
   가능한 구조이지만 이럴땐 난이도가 높은 유형에 속한다. 이런 문제를 푸는 수준이라면...
   그냥... 구글 수석으로 가자!

문제에서 가장 신경써야할 점은
2, 3번 이다!!

## 큰돌님이 알려주는 DP 확인법

1. 완전 탐색으로 되는지 생각해보기
2. 메모이제이션이 필요한지 생각해보기
   a. 100만정도면 DP로
   b. 1억 정도면 그리디

## DP

완전탐색 + 메모이제이션

#### 메모이제이션

맵이든 셋이든 배열이든 계산된 값을 저장해 놓아서 다시 한번 계산하는 것을 방지하는 것이죠.

주로 배열의 형태로 저장을 한다

- 간혼 맵/셋으로도 저장을 한다
